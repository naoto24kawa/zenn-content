## 監視ツール、入れたはいいけど放置してませんか

UptimeRobot、Healthchecks.io、Visualping。個人開発者なら一度は名前を聞いたことがある監視ツールたち。自分も全部入れてみました。結果? **3ヶ月後には全て放置していました。**

「とりあえず監視は入れとこう」と思って、AI に聞いて「これがいいよ」と言われたやつを片っ端から設定した。無料プランでアカウントを作って、URLを登録して、通知先を設定して。「よし、これで安心だ」と思ったのが去年の春。

そこから一度もダッシュボードを開かなかった。

## 放置した結果、何が起きたか

ある日、ユーザーから「データが更新されてないんですけど」と連絡が来ました。

調べてみたら、**毎日動いてるはずの Cron ジョブが3週間前から止まっていた。**

エラーも出ていない。サーバーは正常に動いている。ただ Cron が静かに死んでいた。デプロイのタイミングで crontab が吹き飛んだのか、プロセスが何かの拍子に落ちたのか、原因もよくわからない。

Healthchecks.io を入れていたはずなのに、アラートが来ていたのかすら覚えていない。メールを遡ってみたら、確かに「Check is DOWN」という件名のメールが来ていた。他のメルマガに完全に埋もれていた。

3週間分のデータが欠損。個人開発とはいえ、使ってくれている人がいるサービスでこれはきつい。Cron の怖いところは、**サイト自体は普通に表示されるから気づかない** ことだと思う。

## なぜ放置するのか

この経験を振り返って、自分が監視ツールを放置する理由を整理してみました。同じ状況の人、たぶん多いと思います。

### 1. 英語だとよくわからない

ダッシュボードもアラートも全部英語。設定画面を開いて「Heartbeat Monitoring」「Incident Management」「Escalation Policy」と並んでいると、なんとなく設定したつもりになるけど、**本当に正しく動いているのか自信がない**。

「Check is DOWN」というメールが来ても、それが「Cron が止まった」なのか「メンテナンス中の一時的な通知」なのか、パッと見で判断がつかない。日本語で「Cronジョブ "データ更新" が24時間実行されていません!」と来たら流石に気づく。でも英語だと斜め読みして終わり。

日本語ドキュメントもほぼない。困ったときに調べても英語の記事しか出てこない。設定に不安があっても「まあ動いてるだろう」で放置してしまう。

### 2. ツールがバラバラ

自分の場合、こんな構成になっていました:

- 死活監視 → UptimeRobot
- Cron の実行確認 → Healthchecks.io
- SSL / ドメイン期限 → レジストラからのメール(見逃す)
- Web ページの変更検知 → Visualping

全部別のサービス、別のダッシュボード、別のログイン。SSL やドメインの期限切れに至っては、レジストラからの更新メールを見逃したら終わり。自動更新にしていても、クレカの期限切れで決済が通らず失効するパターンもある。

結局、通知がバラバラの場所から飛んでくるから見逃す。1箇所でまとめて見えないと、そのうち「まあ動いてるだろう」で全部放置になる。

### 3. 通知が来ても気づかない

ここが一番の問題だった。

メールで通知が来ても、GitHub の通知、SaaS のお知らせ、ニュースレターに埋もれて気づかない。自分のメールボックスには1日50通以上メールが来る。監視アラートが1通紛れ込んでも見つけられない。

Slack 連携もできるけど、Slack は仕事用。個人開発の通知を仕事の Slack に流すわけにもいかないし、個人用ワークスペースをわざわざ作っても結局開かない。

**LINE で来てくれたら気づくのに。**

自分の場合、LINE の通知は5分以内に見る。メールは下手したら翌日。Slack は仕事中だけ。監視アラートは「いつ来るかわからない」から、普段の生活で常に目に入る場所に届かないと意味がない。それは LINE だと思う。

なのに LINE 通知に対応してる監視ツールがない。

## 個人開発の監視、理想はこうだと思う

Cron 停止事件の後、自分がほしいと思ったのはこういうものでした:

- **日本語で全部わかる** -- 設定もアラートも日本語。「Cronジョブが24時間実行されていません」と来てほしい
- **1つのダッシュボードで全部見える** -- 死活もCronもSSLもドメインも変更検知も、1箇所で
- **LINE で通知が来る** -- 気づける場所に届く。5分以内に見る場所に
- **安い** -- 個人開発に月 $20 は払えない。$5 くらいがちょうどいい

既存のツールを調べれば調べるほど「なんで日本語のやつ、ないんだろう」と思いました。海外には監視ツールがたくさんある。でも日本語 UI で、LINE 通知ができて、個人開発者の価格帯で使えるものは **1つもなかった**。

## 作ることにした

ないなら作ろう、ということで監視ダッシュボードを開発し始めました。

技術スタックの話や設計判断については別の記事で書く予定ですが、個人開発者が「自分のサービスをちゃんと見守れるツール」を目指しています。

**Manako**(まなこ) という名前にしました。日本語で「眼」という意味です。興味がある方は Waitlist に登録してもらえると嬉しいです。

https://manako.dev

## まとめ

- 監視ツールは入れただけで安心しがち。放置してる人、自分だけじゃないはず
- 英語UIだと「なんとなく設定した」で終わりやすい。アラートも読み飛ばす
- ツールがバラバラだと面倒になって見なくなる。5つのダッシュボードは無理
- 通知はメールや Slack じゃなくて LINE に来てほしい
- 日本語で、1つで、安く使える監視ツールがほしかった。なかったから作ることにした
